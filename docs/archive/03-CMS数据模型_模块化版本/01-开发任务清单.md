# 01 - 开发任务清单

> **阅读时间**: 2 分钟  
> **适用对象**: 项目经理、开发团队

---

## ✅ 开发任务清单

### Phase 1: 核心模型 (优先级: P0)
- [ ] `Media` + `MediaCategory` (媒体库+分类)
- [ ] `Category` (通用分类系统)
- [ ] `ProductSeries` (产品系列)
- [ ] `Product` (产品/SKU)

### Phase 2: 内容模型 (优先级: P1)
- [ ] `Blog` (博客文章)
- [ ] `Application` (应用案例)
- [ ] `FaqItem` (常见问题)
- [ ] `ContactForm` (表单提交)

### Phase 3: 配置模型 (优先级: P1)
- [ ] `SiteConfig` (站点配置)
- [ ] `Navigation` (导航菜单)
- [ ] `GlobalSEO` (SEO配置)

### Phase 4: 软删除与权限 (优先级: P0)
- [ ] 所有内容模型添加 `status` 字段
- [ ] `Media` 模型添加 `status` 字段
- [ ] 禁用所有物理删除操作
- [ ] 实现 RBAC 权限控制

### Phase 5: 首页单列模型（优先级：P2）
- [ ] `HomePage` (单例模型)
---

## 📋 实施说明

### 优先级定义
- **P0 (最高)**: 核心功能，必须在第一阶段完成
- **P1 (高)**: 重要功能，第二阶段完成
- **P2 (中)**: 增强功能，可根据时间安排

### 依赖关系
1. Phase 1 必须先完成（其他模块依赖这些核心模型）
2. Phase 4 的软删除机制应在所有模型完成后统一实施
3. Phase 2 和 Phase 3 可以并行开发
4. Pahse 5 依赖于`Product`,`ProductSeries`,`Media`,`Application`,应在这些模型完成后完成

### 验收标准
每个 Phase 完成后需要：
- [ ] 所有模型通过 Prisma migrate
- [ ] Keystone Admin UI 可正常访问和编辑
- [ ] GraphQL API 测试通过
- [ ] 多语言字段正确配置（见 `02-多语言方案.md`）
